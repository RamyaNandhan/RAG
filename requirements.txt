NLP Retrieval-Augmented Generation (RAG) System
Overview

This project implements a Retrieval-Augmented Generation (RAG) pipeline that enables natural language querying over document collections using a local LLM.

The system processes documents, stores embeddings in a vector database, retrieves relevant information for user queries, and generates context-aware responses.

Features

Document ingestion from PDFs

Text chunking for efficient retrieval

SentenceTransformer embeddings

Chroma vector database

Local LLM inference using Llama.cpp

Fully offline RAG pipeline

Architecture
Documents → Chunking → Embeddings → Vector DB
                        ↑
User Query → Embedding → Retrieval → LLM → Answer
